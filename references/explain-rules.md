# Explain Rules

개념 설명 요청에 대한 응답 규칙이다.

## 원칙

- 전문 용어를 최소화한다
- 실생활 비유로 먼저 감을 잡게 한다
- 예시 코드는 5줄 이내로 핵심만

## 응답 구조

1. **비유** (1-2문장): 일상적인 사물이나 상황으로 개념을 설명한다
2. **한 줄 정의**: 비유 후 정확한 한 줄 설명
3. **예시 코드** (5줄 이내): 가장 단순한 형태로만
4. **공식문서 링크**: `📖 더 알아보기: [링크]`

## 비유 작성 기준

- 누구나 아는 일상 소재를 사용한다 (택배 상자, 서랍, 교통 신호 등)
- 비유가 개념의 핵심 특성을 정확히 반영해야 한다
- 비유가 맞지 않으면 쓰지 않는다

## 예시 코드 기준

- 실행 가능한 최소 코드
- 변수명은 의미있게
- 주석 없이도 이해되는 수준으로
- 5줄 초과 시 핵심 부분만 발췌

## 예시 (Swift Optional)

비유: "Optional은 택배 상자 같아요. 상자 안에 물건이 있을 수도 있고(Some), 없을 수도 있어요(nil). 받기 전에 열어봐야 알 수 있죠."

한 줄 정의: "값이 있을 수도, 없을 수도 있는 타입입니다."

예시 코드:
```swift
var name: String? = "Alice"
if let unwrapped = name {
    print(unwrapped) // "Alice"
}
```

📖 더 알아보기: https://docs.swift.org/swift-book/documentation/the-swift-programming-language/thebasics/#Optionals
