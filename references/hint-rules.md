# Hint Rules

문제 해결 요청에 대한 힌트 3단계 규칙이다.

## 원칙

- 절대 직접 답을 주지 않는다
- 각 단계 후 사용자가 시도할 시간을 준다
- 힌트는 짧고 명확하게 (2-3문장 이내)

## 힌트 단계

### 1단계: 영역 좁히기
어느 부분을 봐야 하는지 방향만 알려준다. 구체적인 원인이나 해결책은 언급하지 않는다.

예시:
- "레이아웃 컨테이너 쪽을 살펴보세요."
- "에러 메시지에서 어떤 타입이 언급되고 있나요?"
- "이 함수가 언제 호출되는지 추적해보세요."

### 2단계: 구체적 단서
1단계보다 구체적인 단서를 준다. 공식문서 링크를 함께 제공한다.

예시:
- "ScrollView는 내부 콘텐츠에 명시적인 크기가 필요할 수 있어요."
- "Optional 바인딩이 실패하는 경우를 생각해보세요."

### 3단계: 거의 정답
사용자가 약간만 더 생각하면 답을 낼 수 있는 수준의 힌트.

예시:
- "`.frame(maxWidth: .infinity)`가 빠진 건 아닐까요?"
- "`if let` 대신 `guard let`을 써보면 어떨까요?"

## 3단계 후에도 못 풀면
직접 설명을 제공한다. 설명 후 공식문서 링크를 함께 제시한다.
